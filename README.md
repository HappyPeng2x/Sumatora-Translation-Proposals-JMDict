# Sumatora Translation Proposals (JMdict)

AI-drafted translations of [JMdict](https://www.edrdg.org/jmdict/j_jmdict.html) senses that are
missing in a given non-English language, generated by
[SumatoraIndex](https://github.com/HappyPeng2x/SumatoraIndex)'s
`find-translation-gaps.py` / `propose-translations.py`, for manual review before being folded
back into the JMdict-derived data used by
[Sumatora Dictionary](https://github.com/HappyPeng2x/SumatoraDictionary).

**These are drafts, not accepted translations.** Nothing here should be treated as a vetted
addition to JMdict until a human has reviewed and corrected it.

## Layout

Each language gets a top-level directory (e.g. `fre/` for French):

```
<lang>/
  <shard>/<seq>.json       one proposed translation per JMdict entry (sharded by seq // 10000)
  chunks/
    chunk-NNNN.jsonl       manifest: one {"seq", "path"} line per entry completed in that window
    chunk-NNNN.done        present once the chunk is closed and safe to consume
    chunk-NNNN.md          human-readable table (headword / English / French) for the same chunk
    .progress.json         internal generator state (current chunk, when it opened)
```

Only chunks with a `.done` marker are guaranteed complete -- the highest-numbered chunk for a
language may still be open and accumulating entries.
